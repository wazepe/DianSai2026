## 📦 简介

基于 **MSPM0G3519** 的电赛小车底层模板，覆盖了竞赛中绝大多数常用外设：

- 时钟系统（`clock.c`）
- BNO080 姿态传感器（UART + DMA）
- 双路编码器（QEI）
- 双路直流电机（PWM + 方向控制）
- 8路灰度传感器（CD4051模拟开关）
- 超声波测距（PWM触发 + 输入捕获）
- 4路按键（单击/双击/长按/连发）
- OLED显示（I2C，支持中文/绘图）
- 蓝牙串口通信（环形缓冲区 + 包解析）

代码已经按模块拆分，每个模块独立可测，适合快速搭建赛道底层。

---

## 🔌 外设速览

| 模块 | 关键实现 |
|------|----------|
| 时钟 | `g_sysTick_1ms_u32` + 阻塞延时 |
| BNO080 | DMA接收19字节，校验和验证，`yaw/pitch/roll` ÷100 |
| 编码器 | QEI直接读数，右通道乘`-1`（机械安装反向） |
| 电机 | `Load(int8_t, int8_t)`，±100范围，仅在变化时更新 |
| 灰度 | 8通道轮询，丢线时保持上次有效值 |
| 超声波 | 捕获脉宽，`计数值×0.17`，>6000归零 |
| 按键 | 20ms扫描，支持单/双/长/连发 |
| OLED | 显存操作，更新时才刷新，避免频繁I2C |
| 蓝牙 | 环形缓冲 + 状态机解析`[...]`格式数据 |
| PID | 位置式，带积分分离和输出限幅 |

## 🚀 快速使用

```bash
git clone https://github.com/你的用户名/电赛小车模板.git
```

1. 使用 SysConfig 配置各外设引脚（详见各模块头文件注释）
2. 根据实际硬件调整 GPIO 映射
3. 编译下载，串口看到 `MSPM0G3519_Init_OK` 即表示启动正常

**注意**：OLED 驱动来自江协科技，已适配 MSPM0 硬件 I2C。

## ⚙️ 关键参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `period` | 8000 | PWM 周期（对应 20kHz 左右） |
| `distVal > 6000` | 60cm | 超声波有效测量上限 |
| `KEY_TIME_LONG` | 2000ms | 长按判定阈值 |
| `环形缓冲区` | 100 字节 | 蓝牙接收缓存 |
| `BNO080 校验和` | 字节2~14 | 与字节18对比，不一致则丢弃 |

## 📌 注意事项

- BNO080 是 **RX only** 模式，只需接 TX → RX，无需双向通信
- 右编码器读数取了反，若你的机械安装正确，请去掉 `* -1`
- 超声波超过 60cm 会归零，实测有效距离约 20~450cm
- 灰度 `delay_cycles(CPUCLK_FREQ / 1000000)` 依赖主频，若修改主频需同步调整
